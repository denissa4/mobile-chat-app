## Мобильное чат-приложение (ChatGPT-style) с интеграцией Azure Bot Service

**Дата:** 13.06.2026  
**Репозиторий:** https://github.com/denissa4/mobile-chat-app

---

## 1. Что было сделано

Разработано мобильное приложение — один экран чата в стиле ChatGPT — с полной интеграцией в Azure Bot Service через протокол Direct Line 3.0.

Проект состоит из двух частей:
- **Backend** — Node.js/Express сервер, защищает секрет Azure
- **Mobile app** — React Native (Expo) приложение

## 6. Ключевые технические решения

### 6.1. Собственный Direct Line WebSocket клиент

**Проблема:** стандартная библиотека `botframework-directlinejs` из ТЗ использует браузерные API (`XMLHttpRequest`, `EventSource`) и не работает в React Native.

**Решение:** написан собственный клиент (`useDirectLine.ts`) на нативном `WebSocket` и `fetch`, полностью реализующий протокол Direct Line 3.0:
1. Получение токена с бэкенда
2. Создание разговора (`POST /v3/directline/conversations`)
3. WebSocket соединение по `streamUrl`
4. Отправка сообщений через REST
5. Приём активностей через WebSocket

### 6.2. Merge-буфер входящих сообщений

**Проблема:** бот разбивает один ответ на несколько отдельных `message`-активностей, которые приходят подряд — они отображались как разные пузыри.

**Решение:** входящие `message`-активности не рендерятся сразу — они накапливаются в буфере и сбрасываются в одно сообщение через 600 мс после последней активности. Умное объединение: если предыдущий фрагмент не заканчивается знаком препинания — фрагменты конкатенируются напрямую (без `\n\n`), чтобы не разрывать слова.

### 6.3. Анимация появления ответов

**Проблема:** анимация внутри `react-native-marked` (который сам использует FlatList) вызывала прыжки текста и визуальные артефакты.

**Решение:** анимация вынесена на уровень компонента `Bubble`. Используется Reanimated v4: `opacity` (0→1, 700 мс, кубический ease-out) + `translateY` (18→0, spring с damping 18, stiffness 100) — плавное появление снизу без конфликтов с внутренним FlatList.

### 6.4. Индикатор «бот печатает»

**Проблема:** бот присылает `typing`-активности, но таймер индикатора (3 сек) истекал раньше чем приходил ответ — экран выглядел пустым.

**Решение:** при поступлении `message`-активности в merge-буфер таймер typing не просто продлевается, а `isTyping` принудительно устанавливается в `true` без нового таймера. Индикатор горит до самого появления сообщения.

### 6.5. Поведение прокрутки (как у ChatGPT)

- **После отправки:** `scrollToIndex` с `viewPosition: 0, viewOffset: -60` — сообщение пользователя поднимается вверх экрана с 60px отступом, ниже пространство для ответа бота.
- **Во время ответа бота:** автопрокрутка не выполняется — пользователь видит анимацию появления ответа в пространстве ниже своего сообщения.
- **Кнопка ↓:** появляется при прокрутке более чем на 120px от конца списка, позволяет вернуться вниз.

### 6.6. Требования к userId

Azure Direct Line требует, чтобы `userId` начинался с префикса `dl_`. Нарушение этого требования приводит к ошибке `400 tokenParameters is missing User`. Реализован генератор: `dl_` + timestamp в base36 + случайная строка.

## 7. Безопасность

- `CHAT_APP_DIRECT_LINE_SECRET` находится только в `backend/.env` (в `.gitignore`)
- Приложение получает краткосрочный токен (TTL 30 минут) через HTTPS
- Токен не сохраняется между сессиями (пересоздаётся при каждом запуске)
- Трафик приложение → бот идёт по WSS (WebSocket Secure)

## 8. Локальный запуск

### Backend

```bash
cd backend
cp .env.example .env
# Вписать CHAT_APP_DIRECT_LINE_SECRET в .env
npm install
node server.js
# Сервер на http://localhost:3000
```

Для тестирования с физического iPhone — запустить ngrok:
```bash
ngrok http 3000
# Скопировать HTTPS URL в chat/.env → EXPO_PUBLIC_API_BASE_URL
```

### Mobile

```bash
cd chat
cp .env.example .env
# Вписать EXPO_PUBLIC_API_BASE_URL (URL бэкенда)
npm install
npx expo start
# Открыть в Expo Go на iPhone
```

## 9. Нерешённые вопросы и ограничения

- **Персистентность диалога (FR-12):** при перезапуске приложения диалог сбрасывается — новая сессия, новый `conversationId`. Сохранение истории не реализовывалось согласно приоритету ТЗ.
- **Стриминг токен-за-токеном:** бот `AI-employee-dev-user-assigned` отдаёт ответ целиком, а не побайтово — реального стриминга нет. Fade-in анимация создаёт визуальный эффект плавного появления.
- **EAS Build:** приложение протестировано в Expo Go. Production-сборка через EAS не выполнялась.
- **Refresh токена:** при истечении токена (`ExpiredToken`) приложение переподключается полностью (новый токен + новый разговор). Тихий refresh в рамках текущего разговора не реализован.
