**Версия ТЗ:** 1.2 (стек зафиксирован окончательно, без альтернатив)

---

## 1. Цель и краткое описание

Разработать мобильное приложение — **один чат в стиле ChatGPT** — на React Native (Expo) с интеграцией в бэкенд через **Azure Bot Service / Direct Line**.

Принципиально: это **не мессенджер с историей**. Приложение состоит из **единственного экрана**:

- один непрерывный диалог, который можно прокручивать;
- **нет** списка чатов, **нет** кнопки «новый чат», **нет** боковой панели/истории;
- современный интерфейс с индикатором «печатает», markdown и плавными анимациями.

---

## 2. Объём работ (Scope)

### Входит в объём
- Мобильное приложение (iOS + Android) на Expo.
- Один экран чата с полем ввода, лентой сообщений, авто-прокруткой.
- Рендер ответа бота в markdown, индикатор печати.
- Интеграция с Azure Direct Line через официальную библиотеку `botframework-directlinejs`.
- Безопасное получение токена Direct Line с бэкенда (секрет не хранится в приложении).

### НЕ входит в объём
- История прошлых диалогов и переключение между чатами.
- Регистрация/логин, профили (если не потребуется — см. Открытые вопросы).
- Push-уведомления.
- Сам бот в Azure (предполагается, что он развёрнут — `AI-employee-dev-user-assigned`).
- Веб-версия (опционально — см. Открытые вопросы).

## 4. Архитектура

```
┌──────────────────────────┐     1) запрос токена (HTTPS)    ┌────────────────────────────┐
│   Expo / RN app          │ ──────────────────────────────► │  Backend (Azure App Service)│
│   один экран чата        │ ◄────── токен Direct Line ────── │  хранит DIRECT_LINE_SECRET, │
│   botframework-directline│                                 │  меняет секрет на токен      │
└───────────┬──────────────┘                                 └────────────────────────────┘
            │  2) прямое WebSocket-соединение с токеном
            ▼
   ┌──────────────────────────────┐     Direct Line 3.0     ┌──────────────────┐
   │  Direct Line (Azure)         │ ◄────── activities ────► │  Бот в Azure     │
   └──────────────────────────────┘                          └──────────────────┘
```

### Ключевое правило безопасности
**Direct Line Secret (`CHAT_APP_DIRECT_LINE_SECRET`) НИКОГДА не попадает в приложение.** Секрет живёт только на бэкенде (Azure App Service, переменные `CHAT_APP_DIRECT_LINE_*`).

### Роль бэкенда (минимальная — только токен)
Один эндпоинт `POST /api/token`: меняет секрет на временный токен Direct Line (`POST .../v3/directline/tokens/generate`) и возвращает его приложению. Мост/прокси не нужен — directlinejs сам открывает WebSocket по токену.

## 6. Нефункциональные требования (NFR)

- **Платформы:** iOS 15+, Android 8+ (свериться с минимумами актуального Expo SDK).
- **Производительность:** плавная прокрутка при 200+ сообщениях; рендер markdown без лагов.
- **Отзывчивость:** первый отклик на отправку < 100 мс (оптимистичный рендер).
- **Типизация:** TypeScript strict; ESLint + Prettier.
- **Доступность:** контраст, тач-зоны ≥ 44pt, тёмная/светлая темы.
- **Безопасность:** секрет только на сервере; токены в `expo-secure-store`; трафик по HTTPS/WSS.

## 9. Интеграция с Direct Line (поток данных)

1. Приложение запрашивает токен (`POST /api/token`). Бэкенд меняет секрет на токен и возвращает.
2. Приложение создаёт `new DirectLine({ token })`.
3. Подписка на `directLine.activity$` — приём сообщений бота (WebSocket, realtime).
4. Отправка через `directLine.postActivity({ type: 'message', from: { id }, text })`.
5. Подписка на `directLine.connectionStatus$` — статусы и refresh токена при `ExpiredToken`.
6. Рендер сообщений и markdown по мере прихода activity.

> **Зависимость от бота:** token-by-token стриминг возможен, только если сам бот стримит (Bot Framework streaming). Если бот отдаёт ответ целиком — мобилка показывает «печатает», затем сообщение. Уточнить у владельца бота.

## 12. Критерии приёмки (Definition of Done)

- Запускается в Expo Go (UI) и в Development/EAS build (полный функционал).
- Открывается сразу на одном экране чата; нет истории, списка чатов, кнопки «новый чат».
- Сообщение уходит в бот через `botframework-directlinejs`, ответ отображается.
- Markdown рендерится корректно, включая код-блоки.
- Индикатор печати, авто-прокрутка, статусы соединения и refresh токена работают.
- Direct Line Secret отсутствует в коде/бандле/репозитории; токен получается с бэкенда.
- В проекте нет Vercel AI SDK и стриминговых полифилов.
- Использованы только зафиксированные в разделе 3 библиотеки.
- TypeScript strict без ошибок; линт зелёный.
